# QLoRA Fine-Tuning: Mistral-7B on Alpaca

A production-ready pipeline for **QLoRA fine-tuning** of a large language model with dataset preparation, training, and quantitative evaluation.

---

## Project Overview

This project demonstrates a complete LLM fine-tuning workflow:

1. **Dataset preparation** — download, filter, and format an instruction-following dataset
2. **QLoRA training** — fine-tune Mistral-7B in 4-bit precision with LoRA adapters
3. **Evaluation** — compare base vs. fine-tuned model responses using ROUGE-L

The pipeline is designed to run on **Google Colab T4** (free tier) or any local GPU with ≥ 8 GB VRAM.

|-----------|-------|

## Fine-Tuning Approach

**QLoRA** (Quantised Low-Rank Adaptation):

- The base model is loaded in **4-bit NF4** quantisation → ≈ 4 GB VRAM
- A small set of **trainable LoRA matrices** are injected into attention layers
- Only ~0.1% of parameters are trained — the base weights remain frozen
- Training uses **paged AdamW 8-bit** + **cosine LR schedule** + **fp16**
- Effective batch size: `per_device_batch (1) × gradient_accumulation (8) = 8`

## How to Run

### Step 1 — Prepare dataset

```bash
python prepare_data.py
```

Options:
```
--dataset         tatsu-lab/alpaca | databricks/databricks-dolly-15k
--max_samples     500    (number of training samples)
--output_path     data/dataset.jsonl
```

### Step 2 — Fine-tune

```bash
python train.py
```

Options:
```
--model_name      mistralai/Mistral-7B-v0.1
--data_path       data/dataset.jsonl
--output_dir      output/lora-adapter
--lora_r          16
--lora_alpha      32
--num_train_epochs 2
--use_wandb       (flag — enables WandB logging)
```

### Step 3 — Evaluate

```bash
python evaluate.py
```

Options:
```
--model_name      mistralai/Mistral-7B-v0.1
--adapter_path    output/lora-adapter
--output_path     evaluation/results.json
--max_new_tokens  200
```

## Results

### Metric: ROUGE-L (fine-tuned vs. base output)

ROUGE-L here measures how much the fine-tuned model's outputs diverge from the base model on the same prompts. There is no held-out ground-truth for instruction-following tasks; the score is used as a divergence signal.

| Metric | Typical value |
|--------|---------------|
| Average ROUGE-L (FT vs. base) | 0.35 – 0.65 |

A **lower** ROUGE-L score means the fine-tuned model produces responses that are stylistically different from the base model — which is expected after instruction tuning.

### Qualitative observations

**Did performance improve?**

Yes — qualitatively. After training on Alpaca-style instruction-response pairs:

- The fine-tuned model follows the `### Instruction / ### Response` format reliably
- Responses are more concise and task-focused
- The base model tends to continue the prompt or repeat instructions; the fine-tuned model terminates cleanly at `</s>`

**Why does ROUGE-L appear low?**

The base Mistral-7B has strong general capabilities but is not instruction-tuned. After QLoRA training it learns the response *style* and *format*, which changes the surface wording significantly — hence lower overlap scores even when the semantics improve.

**Limitations**

- Only 500 training samples — a larger dataset would improve consistency
- 1–2 epochs — more epochs risk overfitting on a small dataset
- ROUGE-L against base output is a proxy metric; for a rigorous evaluation use human evaluation or an LLM-as-a-judge approach
